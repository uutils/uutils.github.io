Return the canonical, absolute form of a path.

This function is a generalization of std::fs::canonicalize that allows controlling how symbolic links are resolved and how to deal with missing components. It returns the canonical, absolute form of a path. The miss_mode parameter controls how missing path elements are handled

  • MissingHandling::Normal makes this function behave like std::fs::canonicalize, resolving symbolic links and returning an error if the path does not exist.
  • MissingHandling::Missing makes this function ignore non-final components of the path that could not be resolved.
  • MissingHandling::Existing makes this function return an error if the final component of the path does not exist.

The res_mode parameter controls how symbolic links are resolved:

  • ResolveMode::None makes this function not try to resolve any symbolic links.
  • ResolveMode::Physical makes this function resolve symlinks as they are encountered
  • ResolveMode::Logical makes this function resolve ‘..’ components before symlinks
